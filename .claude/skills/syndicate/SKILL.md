# syndicate — ROSSE 배포 파이프라인

Repo: `~/repos/gh/memex-kb`. `run.sh`가 *명령*을 덮고, 이 파일이 *판단·규칙·함정*을 덮는다.

**전략 SSOT는 코드가 아니라 노트다** — autholog 노트
`~/sync/org/notes/20250324T110312--힣-원석-날것을-휘갈긴다...autholog....org` (id `20250324T110312`).
"왜 이렇게 하는가"(원석 날것·일일일생·면별 규칙)는 거기서 읽는다. 이 스킬은 "어떻게 실행하는가"만 담는다.
포맷 규격의 **런타임 SSOT는 `scripts/syndicate.py`의 `PLATFORMS` dict** — 표가 의심되면
`./run.sh syndicate-specs`로 코드에서 다시 뽑는다. (구 `docs/syndication.md`는 폐기됨.)

## 멘탈 모델 — ROSSE

POSSE(Post on Own Site, Syndicate Everywhere)를 뒤집은 **R**aw Outside, **S**yndicate via own **S**ite, **E**verywhere.

```
날것 원문 (링크드인/위클리 저널, 손가락 작성)
      ↓  에이전트가 가든으로 옮기고 해설·관련노트 붙임 (봇로그 아님 — 원문이 핵심)
가든 canonical (notes.junghanacs.com, autholog) — 해설+원문 내장   ← SSOT
      ↓  syndicate (이 스킬)
8개 공개면 — 형태는 달라도 전부 가든 링크로 수렴 → 서로를 방어
```

원칙: **편집 없는 원문 + 에이전트 해설의 이중 레이어** · 모든 출력에 **가든 canonical 링크 필수** ·
타겟 맞춤형 호객글·짜집기 글 절대 금지.

## 완전 자동화는 1차 목표가 아니다 (중요)

힣은 **개인 프로필**에 올린다 → 공식 쓰기 API가 막혀 있거나(페북 개인프로필 = Graph API 발행 불가, 2018 폐기 ·
티스토리 Open API = 2024 폐쇄) 없다(네이버 블로그). 그래서 1차 산출물은 **"매체별 복붙용 포맷 묶음 파일 1개"**.
사람이 직접 복붙하거나, **브라우저 클로드가 그 파일을 읽고 각 면에 꽂는다**. API 직접 발행은 후속 후보로만 남긴다.
→ 이 때문에 면별 퍼블리셔가 복붙/브라우저로 수렴 = **스킬도 면별 N개가 아니라 이 하나면 된다.**

## 입력 규약

단순 마크다운 + front matter. 예시 `samples/syndicate-sample.md`.

```markdown
---
title: 제목
garden_url: https://notes.junghanacs.com/notes/<ID>
---

## 해설
가든 canonical 해설 본문 (에이전트가 정리).

## 원문
링크드인 등에 손가락으로 남긴 날것 원문. 편집 없음.

## 요약
### ko
한국어 요약 (요약형 면용).
### en
English summary (블루스카이 등 lang=en 면).
```

## 명령

```bash
./run.sh syndicate <INPUT.md>                  # → out/syndicate/<name>.bundle.md
./run.sh syndicate <INPUT.md> --only threads,bluesky
./run.sh syndicate-specs                        # 매체 명세 표 (코드 SSOT에서 생성)
```

출력 묶음은 매체명 `## 헤딩` + 복붙 펜스 + 함정 메모로 구성. 요약형이 글자수 상한을 넘으면
stdout과 블록 위에 ⚠️ 경고가 찍힌다 → 그 면은 줄여야 한다.

## 워크플로 (한 글을 뿌리는 한 사이클)

1. **원문 입력** — 힣이 링크드인/저널에 남긴 날것을 받는다 (URL 또는 붙여넣기).
2. **가든본 매칭** — `denotecli` / `semantic-memory`로 붙일 기존 노트를 찾는다.
   **새 노트 금지(당분간), 기존 노트 확장.** 전부 `autholog` 태그. 빈방 임시노트·오래된 글·같은 주제 글을 재확장.
3. **입력 md 작성** — 위 규약대로 해설/원문/요약(ko·en) 3블록. 해설=canonical 해설, 원문=편집 없는 날것.
4. **묶음 생성** — `./run.sh syndicate <입력.md>`.
5. **요약형 다듬기** — ⚠️ 경고난 면은 글자수에 맞춰 손본다 (기계 절단 금지 — 매체별로 자연스럽게).
6. **배포** — 사람이 복붙하거나, 브라우저 클로드에게 묶음 파일을 주고 "매체명 헤딩대로 각 면에 넣어라".
   가든 canonical 링크가 모든 면에 들어갔는지 확인.

## 함정 / 판단

- **짜집기 글 금지**: 전문형(네이버·티스토리)은 *반드시* 가든본 전체(원문 내장)여야 한다. 인터넷 짜깁기 본문은 절대 안 내보낸다.
- **원문 불변**: 원문형/전문형의 `## 원문` 블록은 편집하지 않는다. 다듬는 건 해설과 요약뿐.
- **가든 링크 = 수렴점**: 빠지면 ROSSE가 깨진다. 모든 블록에 필수, 인스타는 클릭 불가라 '링크는 프로필' 명시.
- **마크다운 안 먹는 면**: 네이버 스마트에디터는 `##`/`**` 리터럴로 박힌다. 전문형은 그래서 평문 구분선(`─── 원문 ───`)을 쓴다.
- **요약 언어**: 블루스카이는 `en`, 나머지 짧은 면은 `ko` 기본. 입력에 해당 언어 요약이 없으면 ko로 폴백.
- **포맷 규격 추가/변경**: `PLATFORMS` dict 한 곳만 고친다. 이 표는 스냅샷이니 같이 갱신하거나, 코드를 신뢰하고 `syndicate-specs`로 확인.

## 후속 자동화 후보 (1차 이후)

- **API-clean 직접 발행**: Threads(`threads_content_publish` 토큰 이미 보유 → publish 2단계만) · Bluesky(AT Protocol, 앱 패스워드). 읽기/쓰기 둘 다 스킬에 흡수.
- **브라우저 자동화**: 네이버·티스토리·개인 페북 = `browser-tools`(CDP)/크롬확장이 이 묶음 파일을 입력으로 소비.
- 요약형 글자수 자동 적합화 루프(에이전트가 매체별 다듬기).
- 완성 후 힣봇 군단 주입.
