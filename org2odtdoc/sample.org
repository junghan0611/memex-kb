* 1. 개요

이 문서는 Org-mode 한 파일에서 ODT/DOCX 로 변환하는 파이프라인의 **품질 쇼케이스**다.
pandoc 대신 Emacs =org-odt-export= 를 사용하여 다음 이점을 얻는다.

- =reference.odt= 스타일 정확 적용 (단락·문자·표 스타일)
- 한글 캡션 (그림·표)
- 표 헤더 배경색 + 테두리 자동 보정 (=odt_postprocess.py=)
- CSL bibliography — IEEE 번호 인용과 번호순 참고문헌

#+CAPTION: 문서 메타
| 항목     | 값                                  |
|----------+-------------------------------------|
| 양식     | 정부 R&D 기획보고서 류 문서          |
| 인용     | IEEE 번호 스타일 ([1], [2] …)        |
| 출력     | ODT, DOCX                           |
| 빌드     | =./run.sh build=                    |

* 2. 본문 요소

** 2-1. 표 — 캡션과 헤더 스타일

표는 자동 변환되며, 헤더 행 배경색과 테두리는 =odt_postprocess.py= 가 보정한다.

#+CAPTION: 변환 도구 비교
| 항목         | Pandoc      | Emacs org-odt-export       |
|--------------+-------------+----------------------------|
| 스타일 적용  | 제한적      | =reference.odt= 완벽 매핑  |
| 한글 캡션    | ✗           | ✓ (그림·표)                |
| 표 스타일    | 기본만      | 헤더 배경 + 테두리 보정    |
| 번호 인용    | 설정 필요   | CSL 로 [1],[2] 자동        |

** 2-3. 인용 — IEEE 번호 스타일

본문에서 =[cite:@key]= 로 인용하면 =[1]= 처럼 번호로 렌더링되고,
문서 끝 참고문헌이 번호순으로 정리된다.

- Server-Driven UI 는 Airbnb 에서 실전 검증된 접근이다 [cite:@airbnb-sdui-2021].
