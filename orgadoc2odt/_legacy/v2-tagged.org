#+HWPX: 25
비용 절감으로 물리센서 구매·설치·유지보수 비용(센서당 2~5만원)을 절감할 수 있음. 둘째, 서비스 확장성으로 기존 센서만으로 새로운 센싱 기능을 추가하여 센싱 종류를 2~3배 확장할 수 있음. 셋째, 센서 자가치유로 물리센서 고장 시 소프트센서가 대체값을 생성하여 서비스 가용성 99% 이상을 유지함. 넷째, 프라이버시 보호로 온디바이스 추론을 통해 데이터 외부 전송 없이 클라우드 유출 위험을 원천 차단함. 다섯째, 저지연 응답으로 네트워크 왕복 없이 50ms 이하의 실시간 응답이 가능함.

#+HWPX: 31
데이터처리 플랫폼은 4개 계층으로 구성됨.

#+HWPX: 32
L1(데이터 수집 계층)은 Matter Controller가 Matter 디바이스와 통신하여 센서 값을 수신하고, Zigbee Bridge가 1차년도 Zigbee 센서 데이터를 수집하며(Zigbee2MQTT 활용), External API Client가 기상청·일출/일몰 등 외부 데이터를 수집함.

#+HWPX: 33
L2(데이터 처리 계층)은 Time Synchronizer가 이기종 센서 데이터의 시간을 정렬하고(NTP 기반), Data Normalizer가 센서 값 정규화·단위 변환·이상값 필터링을 수행하며, Feature Extractor가 AI 입력용 특징 벡터를 생성함.

#+HWPX: 34
L3(AI 추론 계층)은 sLLM Engine이 소형언어모델 기반 추론을 수행하고(Optimium SDK, NPU 가속), Model Manager가 모델 로딩·버전 관리·업데이트를 담당하며(ONNX Runtime, TFLite), Inference Scheduler가 추론 주기와 배치 처리를 관리함.

#+HWPX: 35
L4(출력 계층)은 Matter Bridge가 소프트센서를 Matter Cluster로 노출하고, Local Dashboard가 웹 기반 모니터링 UI를 제공하며, Cloud Connector가 선택적 클라우드 연동(백업, 원격 모니터링)을 지원함.

#+HWPX: 51
온디바이스 AI 구현을 위해 다음 기술들을 적용함. 모델 경량화는 Pruning, Quantization, Distillation 기법으로 모델 크기를 축소하며, 본 과제에서는 INT8 양자화와 Knowledge Distillation을 적용함. NPU 가속은 신경망 전용 프로세서를 활용하여 추론 속도를 높이며, Optimium SDK로 NPU 최적화를 수행함. 모델 최적화는 타겟 하드웨어에 맞춘 연산 최적화로 ONNX Runtime과 TensorRT를 적용함. 메모리 관리는 제한된 RAM에서 효율적 추론을 위해 메모리 풀링과 동적 할당을 최적화함. 전력 효율은 배터리/상시 전원 환경에 대응하여 저전력 추론 모드를 지원함.

#+HWPX: 57
Application Layer는 Device Types와 Clusters로 디바이스 기능(센서, 조명 등)을 정의하며, 본 과제에서는 소프트센서를 Sensor Device Type으로 정의함. Data Model은 Attributes, Commands, Events로 데이터 구조 및 제어 명령을 정의하며, TemperatureMeasurement, OccupancySensing 등 Cluster를 사용함. Interaction Model은 Read, Write, Subscribe, Invoke로 디바이스 간 상호작용 방식을 정의하며, Subscribe로 실시간 센서 값을 전달함. Security는 CASE, PASE, ACL로 인증·암호화·접근 제어를 담당하며, Matter 인증 보안 요구사항을 준수함. Transport Layer는 IPv6, TCP/UDP로 네트워크 통신을 담당하며, Thread와 Wi-Fi 기반 통신을 사용함.

Matter는 5개 계층으로 구성됨. Application Layer는 Device Types와 Clusters로 디바이스 기능(센서, 조명 등)을 정의하며, 본 과제에서는 소프트센서를 Sensor Device Type으로 정의함. Data Model은 Attributes, Commands, Events로 데이터 구조 및 제어 명령을 정의하며, TemperatureMeasurement, OccupancySensing 등 Cluster를 사용함. Interaction Model은 Read, Write, Subscribe, Invoke로 디바이스 간 상호작용 방식을 정의하며, Subscribe로 실시간 센서 값을 전달함. Security는 CASE, PASE, ACL로 인증·암호화·접근 제어를 담당하며, Matter 인증 보안 요구사항을 준수함. Transport Layer는 IPv6, TCP/UDP로 네트워크 통신을 담당하며, Thread와 Wi-Fi 기반 통신을 사용함.
